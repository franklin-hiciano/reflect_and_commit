The Architecture Core BlueprintThe Problem: Direct orchestration from cloud platforms (like Modal) trips enterprise-grade anti-bot mitigations instantly and risks execution timeouts during long background runs.  The Solution: Decouple reasoning from execution. The cloud handles the expensive cognitive load (thinking, DOM parsing, state tracking), while the local laptop acts as an automated input device.  How to Delegate to Sub-Agents: If you are dividing this task across individual coding models, paste Document 0 as global context, then feed each model its respective Phase Document (1 through 5) in isolation. Rust Daemon Manager Layout (src-tauri/src/main.rs)Rustuse std::process::{Command, Child};
use tauri::{State, Manager, RunEvent};
use std::sync::Mutex;

struct DaemonState {
    orchestrator_process: Mutex<Option<Child>>,
}

#[tauri::command]
fn start_bridge_daemon(session_id: String, state: State<'_, DaemonState>) -> Result<(), String> {
    let child = Command::new("python")
        .arg("local_orchestrator.py")
        .arg("--session")
        .arg(session_id)
        .spawn()
        .map_err(|e| e.to_string())?;
        
    *state.orchestrator_process.lock().unwrap() = Some(child);
    Ok(())
}
2. Critical IPC and Resource LifecycleZombie Prevention: You must intercept system interrupts to avoid leaving rogue python or playwright processes dangling on the user's operating system.Tauri Lifecycle Hook: Register an explicit drop handler inside Tauri’s run loop:Rusttauri::Builder::default()
    .manage(DaemonState { orchestrator_process: Mutex::new(None) })
    .build(tauri::generate_context!())
    .expect("error while building tauri application")
    .run(|app_handle, event| match event {
        RunEvent::ExitRequested { .. } => {
            let state = app_handle.state::<DaemonState>();
            if let Some(mut child) = state.orchestrator_process.lock().unwrap().take() {
                let _ = child.kill(); // Force terminate background processes cleanly
            }
        }
        _ => {}
    });
